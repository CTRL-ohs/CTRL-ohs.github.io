Missing secpol.msc

If you're unable to access secpol.msc, you may need to reinstall it. First, you should check whether you can access local policies through gpedit.msc. Open gpedit, then navigate to Computer Configuration > Windows Settings > Security Settings. This is effectively the same as opening secpol.msc, and contains the same options.

If that doesn't work, you may need to reinstall gpedit. To do so, copy the following commands into a txt file, and save it as a .bat file. Run this file as an administrator, and it will install gpedit.msc.

@echo off
pushd "%~dp0"
dir /b %SystemRoot%\\servicing\\Packages\\Microsoft-Windows-GroupPolicy-ClientExtensions-Package~3*.mum >Files.txt
dir /b %SystemRoot%\\servicing\\Packages\\Microsoft-Windows-GroupPolicy-ClientTools-Package~3*.mum >>Files.txt
for /f %%i in (\'findstr /i . Files.txt 2^>nul\') do dism /online /norestart /add-package:"%SystemRoot%\\servicing\\Packages\\%%i"
pause
